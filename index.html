For the technical reader: how it works
1
Split into clausesThe pasted text is broken into individual sentences and de-duplicated.
2
Semantic matchingEach clause is embedded with a MiniLM sentence-transformer and compared (cosine similarity) against labelled examples of fair vs unfair clauses.
3
GDPR grounding (RAG)A FAISS vector index over EU GDPR text retrieves the most relevant article for each flagged clause, so a concern is tied to a real legal basis, not a guess.
4
LLM verification (optional)When an API key is configured, Claude Haiku double-checks each flagged clause to remove false positives. Without it, the tool runs in matching-only mode.

Stack: Python, Flask, sentence-transformers (MiniLM), FAISS, Anthropic API. Built by Taiwo Alabi for an MSc in Artificial Intelligence.
